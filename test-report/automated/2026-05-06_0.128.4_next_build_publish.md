## Notes

- Linux required the explicit musl linker `x86_64-linux-musl-gcc`; the workflow
  now exports `CARGO_TARGET_X86_64_UNKNOWN_LINUX_MUSL_LINKER` for that lane.
- Linux bundled `bwrap` avoids glibc `libcap.a` during musl builds by using a
  small local syscall shim for the cap functions required by bundled
  bubblewrap.
- After the report file was added, the shim received an explicit `<stddef.h>`
  include and `codex-bwrap` was rebuilt successfully. The rebuilt local `bwrap`
  sha256 was `1d7574d17f6c9c610fcf0731a9288bc0637bf28fbeebc43b58e94c62eb7152d4`;
  the published Linux npm tarball hash above remains the one used for npm
  publish verification.
- Android was started only after the Linux build and Linux npm package checks
  were green.
- macOS was not built on MGM. The published darwin package is a source-build
  payload intended for local Mac install/test.
