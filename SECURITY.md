# Security Policy

## Reporting a Vulnerability

For security disclosures, please email **security@mmmbuto.com**.

Please do not file public GitHub issues for security-related problems.

Acknowledgement target: 72 hours. Status update target: 7 days from acknowledgement.

## Supported Versions

Security fixes target the latest released version published on the `main` branch
and the most recent release published on npm / GitHub Releases. Older versions
are not patched separately.

## Scope

This policy covers Codex VL-specific code: the Vivling, `/loop`, `/vl`, `/goal`
workflow commands, the musl/Android build patches, and the Codex VL packaging
on npm (`@mmmbuto/codex-vl`).

For security issues in upstream OpenAI Codex behavior unaffected by Codex VL
additions, please refer to the
[OpenAI Bugcrowd program](https://bugcrowd.com/engagements/openai).
