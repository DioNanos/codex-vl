# Codex VL 0.133.0

Codex VL 0.133.0 updates the fork to the upstream Codex 0.133.0 release line
and keeps the existing fork packaging and install paths aligned with
`@mmmbuto/codex-vl`.

## Upstream Codex 0.133.0

- Goals are now enabled by default, backed by dedicated storage, and track
  progress across active turns. `create_goal`, `update_goal`, and
  `get_goal` are exposed as model tools.
- `codex remote-control` now runs like a foreground command, waits for
  readiness, reports machine status, and keeps explicit daemon-style
  `start` / `stop` commands.
- Permission profiles gained list APIs, inheritance, managed
  `requirements.toml` support, runtime refresh behavior, and stronger
  Windows sandbox integration.
- Plugin discovery is easier to inspect, with marketplace-aware list
  output, installed versions, visible marketplace roots, and remote
  collection support.
- Extensions can observe more lifecycle events, including subagent
  start / stop, tool execution, turn metadata, and async approval / turn
  processing.

Full upstream release notes:
https://github.com/openai/codex/releases/tag/rust-v0.133.0

## Preserved fork features

- `/vivling`, `/vl`, `/loop`, and `/remote-control` slash commands.
- Vivling Memory V2 with V10 schema, bond mechanics, brain expression
  channel, daily LLM budget gating, language-aware boot, Ctrl+J chat
  panel, and CRT footer.
- Fork-safe managed-update channel; the standalone auto-updater stays
  disabled (the daemon never fetches the upstream installer script).
- Fork feedback channel points to `DioNanos/codex-vl/issues`; the
  announcement tip source is `DioNanos/codex-vl/main`.
