# Codex VL 0.132.0

Codex VL 0.132.0 updates the fork to the upstream Codex 0.132.0 release line
and keeps the workflow layer focused on local orchestration, Vivling, and
side-by-side packaging.

## Codex VL Highlights

- `/remote-control` is available inside the TUI for daemon `status`, `start`,
  `stop`, and `restart`.
- `/loop` keeps backward-compatible prompt jobs while adding a structured
  payload foundation for future internal actions.
- `/goal` keeps completed goals clear after completion while blocked goals stay
  inspectable.
- Vivling registry and runtime command internals are split behind stable
  facades, keeping `/vivling` behavior intact while reducing future merge
  friction.
- Public install, update, daemon, doctor, npm wrapper, and source-build
  guidance stay on `@mmmbuto/codex-vl` and `DioNanos/codex-vl`.

## Packages

- Linux x64 native npm package.
- Termux Android arm64 native npm package.
- macOS arm64 source-build npm package.

## Upstream Base

This release incorporates upstream Codex 0.132.0, including Python SDK auth and
turn API updates, `exec resume --output-schema`, batched TUI startup probes,
remote executor auth updates, and app-server image fidelity work.
